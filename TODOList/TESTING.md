# Tempo テスト手順

このファイルは、機能追加や不具合修正のたびに確認する内容を記録するものです。

## 自動テスト

`TODOList` フォルダで次を実行します。

```powershell
node --check task-core.js
node --check script.js
node --test task-core.test.js ui-contract.test.js
git diff --check
```

### `task-core.test.js` で確認すること

- 旧データの読み込みとデータ形式の補正
- プロジェクト、タグ、優先度、期限、ステータスの処理
- プロジェクト削除時の未分類への移動と、タグの追加・名称変更・削除
- 2条件の並び替えと各種グルーピング
- リストとテーブルのプロジェクト・タグ・優先度グルーピング
- 期限切れ・今日・明日・今後7日の判定と期限文言
- ステータス、優先度、プロジェクト、タグ、工数、完了推移の集計
- 保存フィルターと「今日・明日・今後7日」などの期限条件
- 繰り返しタスク、サブタスク、工数、時間計測
- CSVの解析・取込・出力とJSONバックアップの往復
- CSV全文とExcelコピーのタブ区切り貼り付け取込
- 不正値、空文字、重複データを安全に扱えること

### `ui-contract.test.js` で確認すること

- JavaScriptが参照する画面要素がHTMLに存在すること
- ボタン、入力欄、ダイアログの役割と閉じ方
- リスト、テーブル、カンバン、タイムライン、近日予定カレンダーの表示契約
- プロジェクト、お気に入り、保存フィルター、ゴミ箱の操作入口
- タグ管理画面、プロジェクト削除、アプリアイコンとファビコンの参照
- ライト・ダーク・スマートフォン用CSSと、非表示要素が残らないこと
- CSV・JSON操作とWindows向けキーボードショートカット

## ブラウザ操作テスト

自動テスト後、ローカル確認用サーバーで画面を開き、次を実際に操作します。確認用データは通常利用中の `file://` 版と分け、`localhost` 側で扱います。

1. 「すべて」でページを上下へ動かし、サイドバー内も先頭からデータ操作までスクロールできる。
2. タスクが0件のプロジェクトを開き、空表示だけが出て、追加したタスクへ表示中のプロジェクトが付く。
3. 保存フィルターの期限で「明日」を選び、明日のタスクだけになる。
4. 今日・すべて・完了で、プロジェクト別・タグ別・優先度別・グループなしを切り替える。
5. タスク詳細を開き、背景・×・キャンセル・Escで安全に閉じられる。
6. 時間計測を開始・停止し、秒表示と実績時間への加算を確認する。
7. 近日予定で前週・今日・次週へ移動し、タスク詳細を開ける。
8. プロジェクトのお気に入り登録・解除と、サイドバー上部への表示を確認する。
9. ライト・ダークの両方で文字、タスク、タグ、期限の色が読める。
10. 開発者コンソールにJavaScriptエラーがない。
11. 期限サマリーの4区分が重複せず、タスクの期限文言と強調色が一致する。
12. レポートでステータス円グラフ、優先度・プロジェクト・タグの棒グラフ、工数、直近7日の完了グラフが表示される。
13. タスクがあるプロジェクトを削除すると確認が出て、タスクが「未分類」へ移り、お気に入りと保存フィルターも整理される。
14. タグ管理で未使用タグの追加、使用中タグの名称変更と削除を行い、タスク詳細と保存フィルターへ反映される。
15. 近日予定カレンダーで完了タスクがチェック、取り消し線、完了色で区別され、詳細を開ける。
16. プロジェクトを開くと期限サマリーがそのプロジェクトだけの件数になり、「すべて」へ戻ると全体件数へ戻る。
17. ブラウザータブにファビコンが表示され、インストール用マニフェストから192px・512pxのアイコンを読み込める。
18. テーブル表示でプロジェクト別・タグ別・優先度別・グループなしを切り替え、各見出しと件数が正しい。
19. クイック追加欄でプラスの飾りがなく、入力・Enter追加・詳細追加の役割が迷わず判別できる。
20. 「CSV貼付」でカンマ区切りCSVとExcelからコピーしたタブ区切り表を貼り、プレビュー・列割り当て・重複判定を経て取り込める。
21. Windows Edge相当の画面幅で、サイドバーに横スクロールが出ず、縦スクロールがボタンへ重ならない。グループテーブルの横スクロールは1本だけで、右端の「詳細」が常に押せる。
22. テーブルの完了ボタンが未完了時は丸い輪、完了時は緑のチェックとして表示され、ホバー・キーボードフォーカス・ダークモードでも状態を判別できる。
23. タスクがあるプロジェクトを開き、リスト・テーブルの両方でタグ別・優先度別・グループなしを切り替え、表示中プロジェクトのタスクだけが正しくグループ化される。
24. 1920px幅で通常リストが過度に中央へ寄らず左側から始まり、テーブル・カンバン・タイムライン・近日予定カレンダーが本文領域の横幅をほぼ全面利用する。760px以下では従来どおり左右16pxの余白になる。
25. 未完了と完了が混在するプロジェクトを開き、進捗率が「完了数 ÷ そのプロジェクトの全タスク数」で表示され、別プロジェクトや今日のタスクが混ざらない。
26. テーブルで長いタスク名が折り返され、文字数に合わせて行の高さも伸び、途中で見切れない。Enterで編集を確定でき、マウスを重ねると全文も確認できる。

## 変更時のルール

- 共通処理を変更したら `task-core.test.js` に正常系と境界値を追加します。
- HTML、CSS、操作入口を変更したら `ui-contract.test.js` を更新します。
- スクロール、フォーカス、ドラッグ、ダイアログなど画面操作に関わる変更は、ブラウザでも確認します。
- テストがすべて成功してからコミットし、プルリクへ反映します。
