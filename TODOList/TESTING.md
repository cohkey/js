# Tempo テスト手順

このファイルは、機能追加や不具合修正のたびに確認する内容を記録するものです。

## 自動テスト

`TODOList` フォルダで次を実行します。

```powershell
node --check task-core.js
node --check script.js
node --test task-core.test.js ui-contract.test.js
git diff --check
```

### `task-core.test.js` で確認すること

- 旧データの読み込みとデータ形式の補正
- プロジェクト、タグ、優先度、期限、ステータスの処理
- 2条件の並び替えと各種グルーピング
- 保存フィルターと「今日・明日・今後7日」などの期限条件
- 繰り返しタスク、サブタスク、工数、時間計測
- CSVの解析・取込・出力とJSONバックアップの往復
- 不正値、空文字、重複データを安全に扱えること

### `ui-contract.test.js` で確認すること

- JavaScriptが参照する画面要素がHTMLに存在すること
- ボタン、入力欄、ダイアログの役割と閉じ方
- リスト、テーブル、カンバン、タイムライン、近日予定カレンダーの表示契約
- プロジェクト、お気に入り、保存フィルター、ゴミ箱の操作入口
- ライト・ダーク・スマートフォン用CSSと、非表示要素が残らないこと
- CSV・JSON操作とWindows向けキーボードショートカット

## ブラウザ操作テスト

自動テスト後、ローカル確認用サーバーで画面を開き、次を実際に操作します。確認用データは通常利用中の `file://` 版と分け、`localhost` 側で扱います。

1. 「すべて」でページを上下へ動かし、サイドバー内も先頭からデータ操作までスクロールできる。
2. タスクが0件のプロジェクトを開き、空表示だけが出て、追加したタスクへ表示中のプロジェクトが付く。
3. 保存フィルターの期限で「明日」を選び、明日のタスクだけになる。
4. 今日・すべて・完了で、プロジェクト別・タグ別・優先度別・グループなしを切り替える。
5. タスク詳細を開き、背景・×・キャンセル・Escで安全に閉じられる。
6. 時間計測を開始・停止し、秒表示と実績時間への加算を確認する。
7. 近日予定で前週・今日・次週へ移動し、タスク詳細を開ける。
8. プロジェクトのお気に入り登録・解除と、サイドバー上部への表示を確認する。
9. ライト・ダークの両方で文字、タスク、タグ、期限の色が読める。
10. 開発者コンソールにJavaScriptエラーがない。

## 変更時のルール

- 共通処理を変更したら `task-core.test.js` に正常系と境界値を追加します。
- HTML、CSS、操作入口を変更したら `ui-contract.test.js` を更新します。
- スクロール、フォーカス、ドラッグ、ダイアログなど画面操作に関わる変更は、ブラウザでも確認します。
- テストがすべて成功してからコミットし、プルリクへ反映します。
